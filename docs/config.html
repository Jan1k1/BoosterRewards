Configuration (config.yml)

The config.yml file controls the core behavior of the plugin, including database settings, reward definitions, and synchronization timings.

Database Settings

database:
  # Type of database: H2, MYSQL, or MARIADB
  type: H2
  
  # Database connection details (only for MySQL/MariaDB)
  host: "localhost"
  port: 3306
  database: "boosterrewards"
  username: "root"
  password: ""
  
  # Connection pool settings
  pool-size: 10
  connection-timeout: 5000
Recommended: Use H2 for single servers (no setup needed). Use MYSQL or MARIADB for BungeeCord/Velocity networks to sync data across servers.

Rewards Configuration

Define what happens when players boost your server.

rewards:
  # Unique identifier for the reward tier
  booster:
    # Enable or disable this tier
    enabled: true
    
    # Permission required to receive this reward (optional)
    permission: "boosterrewards.booster"
    
    # Commands to run when player starts boosting
    on-boost:
      - "lp user %player% parent add booster"
      - "give %player% diamond 5"
      - "eco give %player% 5000"
      - "msg %player% &dThanks for boosting!"
    
    # Commands to run when boost expires
    on-stop:
      - "lp user %player% parent remove booster"
      - "msg %player% &cYour boost has expired."
    
    # Run commands only once per player?
    one-time: false

Command Prefixes

  • console: (or no prefix) - Executes command as console
  • player: - Executes command as the player
  • msg: - Sends a message to the player
  • broadcast: - Broadcasts a message to the server

Synchronization

sync:
  # How often to check for boost status changes (in seconds)
  interval: 300
  
  # Remove rewards immediately when boost expires?
  remove-rewards-on-stop: true
  
  # Check status when player joins?
  check-on-join: true
  
  # Force sync on server startup?
  verify-on-startup: true

Linking Settings

linking:
  # Linking mode: DISCORD_TO_MINECRAFT or MINECRAFT_TO_DISCORD
  mode: DISCORD_TO_MINECRAFT
  
  # How long codes are valid (in seconds)
  code-expiry: 300
  
  # Length of the linking code
  code-length: 6
  
  # Allow players to unlink and relink?
  allow-relink: true
